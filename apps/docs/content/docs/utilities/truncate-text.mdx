# truncateText

The `truncateText` utility function truncates text to a specified length and adds an ellipsis (...)
to indicate that the text has been shortened. It's perfect for displaying previews, card
descriptions, or any text that needs to fit within specific UI constraints.

## Usage

```javascript
import { truncateText } from './utils/truncateText';

const longText = 'This is a very long text that needs to be shortened';
const shortened = truncateText(longText, 20);
console.log(shortened); // "This is a very long..."
```

## API Reference

### Parameters

- `text` (string): The text to truncate
- `maxLength` (number): The maximum length of the text

### Returns

- `string`: The truncated text with ellipsis if needed

## Examples

```javascript
function BlogCard({ post }) {
  return (

    </div>
  );
}
```

### Comment Previews

```javascript
function CommentPreview({ comment, expanded, onToggle }) {
  const previewLength = 80;

  return (
    <div className="comment-preview">
      <div className="comment-header">
        <span className="author">{comment.author}</span>
        <span className="date">{formatDate(comment.date)}</span>
      </div>
      <p className="comment-text">
        {expanded ? comment.text : truncateText(comment.text, previewLength)}
      </p>
      {comment.text.length > previewLength && (
        <button onClick={onToggle} className="toggle-button">
          {expanded ? 'Show less' : 'Show more'}
        </button>
      )}
    </div>
  );
}
```

## Notes

- The function preserves the original text if it's shorter than the specified length
- Ellipsis (...) is only added when text is actually truncated
- Consider using the `title` attribute to show the full text on hover
- For word-boundary truncation, consider extending the function to avoid cutting words
- The function handles empty strings and null values gracefully
- Useful for maintaining consistent UI layouts with varying content lengths
